Overall

The objective of this is to have a language similar to brainF, but have a math and canvas extention.

+ or a

Add one to the current buffer.

- or s

Subtract one from the current buffer.

l or {Less Than}

Move one buffer to the left, if the buffer is the first one then you move to the last one.

r or {Greater Than}

Move one buffer to the right, if the buffer is the last one then you move to the first one.

~

Copies the buffer index into a variable to be accessed later

=

Copies the buffer index that was copied into the one that is targetted currently.

()

Creates a function at the targetted buffer

& or R

Runs the function at the current spot

[]

While loop that keeps going until the current index is 1

#

Add the copied buffer's value to the current one

?

Subtract the copied buffer's value to the current one

=

Pastes the value from the copied buffer into the current one

$

Clears the buffer

^

Does a trig function based on two parts. The identifying number and the input. The identifying number is a number 0 - 2. 0 = Sine, 1 = Cosine, 2 = Tangent. This number is the value of the copied buffer index. The second value is the input, which is the current index. That is the value you put inside of that function. The output is the index to the very right of that.
